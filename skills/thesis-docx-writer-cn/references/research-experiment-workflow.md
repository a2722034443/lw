# 调研、数据集与实验工作流

## 题目分析

1. 抽取研究对象、任务类型、应用场景和可评价目标。
2. 判断是否需要数据集、系统实现、算法实验、用户测试或工程测试。
3. 形成可验证问题，而不是只写宏观意义。

## 文献检索

优先来源：

- DOI 解析页。
- 期刊官网。
- 知网、万方、维普等数据库。
- IEEE、ACM、Springer、ScienceDirect、MDPI、Frontiers 等出版社或期刊页。
- arXiv 只作为预印本来源，不能伪装成期刊论文。

记录字段：

- 作者。
- 题名。
- 年份。
- 来源。
- 卷、期、页码。
- DOI 或 URL。
- 与论文的使用位置。

## 数据集检索

优先来源：

- 官方数据平台。
- Kaggle。
- UCI。
- Hugging Face。
- Papers With Code。
- GitHub 原仓库。
- 政府、医院、农业、气象等公开数据平台。

记录字段：

- 数据集名称。
- 来源 URL。
- 许可或使用限制。
- 样本量。
- 字段说明。
- 缺失值情况。
- 是否需要注册或授权。
- 适合本题目的原因。

## 实验设计

实验前写清：

1. 数据来源和版本。
2. 预处理步骤。
3. 训练集、验证集、测试集划分。
4. 随机种子。
5. 模型或方法。
6. 超参数。
7. 评价指标。
8. 硬件和软件环境。

不能运行实验时，只能写“实验设计”或“预期验证方法”，不能写虚假指标。

## 结果记录

每次实验至少保存：

- 原始数据来源。
- 预处理脚本或说明。
- 训练脚本或命令。
- 指标结果。
- 输出图表。
- 失败原因或异常日志。

论文正文中写结果时，要和记录一致。

## 实验章节写法

推荐顺序：

1. 实验环境。
2. 数据集说明。
3. 评价指标。
4. 对比方法。
5. 实验结果。
6. 结果分析。
7. 局限性。

结果分析要具体说明哪个指标支持哪个结论，不写“效果显著”“性能优越”这类无依据判断。

## 不可编造规则

1. 未下载的数据集不能写成已使用。
2. 未运行的模型不能写具体指标。
3. 文献没有查到页码不能自己补页码。
4. GitHub 项目不能直接写成学术论文。
5. 公开数据许可不明时要说明限制。
