# 项目型中文论文框架、版式与规模规则

## 必须先联网确认

开始写作或排版前，必须检索并记录当次使用的规则来源。优先级如下：

1. 用户提供的学校模板、学院通知、导师要求。
2. 学校当年本科毕业设计说明书或学位论文格式要求。
3. GB/T 7713.1 学位论文编写规则、GB/T 7714 信息与文献参考文献著录规则。
4. 目标期刊、会议或学院补充规范。
5. 通用本科项目论文写法。

推荐检索式：

- `学校名 本科毕业设计 论文 格式 要求 正文 宋体 小四 首行缩进`
- `学校名 毕业设计说明书 图题 表题 参考文献 格式`
- `GB/T 7713.1 学位论文 编写规则 结构 摘要 目录 图表`
- `GB/T 7714-2015 顺序编码制 参考文献 著录规则`
- `计算机 本科毕业设计 需求分析 系统设计 系统实现 系统测试`
- `软件工程 项目论文 需求分析 总体设计 详细设计 测试 论文结构`

如果没有学校模板，默认按“常见本科毕业设计说明书”处理；如果后续拿到模板，必须以模板覆盖通用规则。

## 项目型论文推荐框架

项目型论文不能只写“用了什么技术”，要从问题、方案、实现、验证形成闭合论证。默认结构如下：

1. 摘要、关键词、英文摘要、英文关键词。
2. 第1章 绪论：研究背景、研究意义、国内外研究现状、问题分析、研究内容、技术路线、论文结构。
3. 第2章 相关技术与理论基础：核心算法、系统框架、数据库、评价指标、关键技术对比。
4. 第3章 需求分析与可行性分析：业务需求、功能需求、非功能需求、角色与用例、数据需求、可行性、风险与制约因素。
5. 第4章 系统总体设计：架构设计、模块划分、业务流程、数据库概念设计、接口设计、安全设计。
6. 第5章 系统详细设计与实现：关键模块实现、核心算法实现、数据库实现、页面或接口实现、异常处理。
7. 第6章 实验、测试与结果分析：实验环境、数据集、评价指标、测试用例、功能测试、性能测试、对比实验、结果讨论。
8. 第7章 工程管理、经济决策与社会责任：进度安排、成本估算、方案选择、法律伦理、隐私安全、环境与可持续性。
9. 第8章 总结与展望：完成工作、主要结果、不足、后续改进。
10. 参考文献、致谢、附录。

若学校模板章节较少，可以合并第7章内容，但不能完全缺失工程管理、经济决策和社会责任论述。

## 2万字论文的规模分配

当用户要求不少于20000字时，按中文正文净字数控制，不把参考文献、目录、代码附录、图片文字重复计入主要正文。默认分配：

| 部分 | 建议字数 |
|---|---:|
| 摘要与关键词 | 600-900 |
| 第1章 绪论 | 2600-3200 |
| 第2章 相关技术与理论基础 | 2600-3200 |
| 第3章 需求分析与可行性分析 | 2600-3200 |
| 第4章 系统总体设计 | 2800-3400 |
| 第5章 详细设计与实现 | 3600-4600 |
| 第6章 实验、测试与结果分析 | 2600-3600 |
| 第7章 工程管理与社会责任 | 1200-1800 |
| 第8章 总结与展望 | 900-1300 |

写作时不要为了凑字数堆概念。扩展字数应优先来自：真实需求场景、方案对比、模块细节、数据表字段解释、测试用例、实验结果分析、风险约束和改进讨论。

## 40张图的规划方式

40张图必须服务论文结构，不能把页面截图无序堆叠。项目型论文可按以下比例规划：

| 章节 | 图件类型 | 建议数量 |
|---|---|---:|
| 第1章 | 技术路线图、研究流程图 | 1-2 |
| 第2章 | 算法流程图、技术原理图、指标解释图 | 4-6 |
| 第3章 | 用例图、活动图、业务流程图、数据流图 | 6-8 |
| 第4章 | 系统架构图、模块结构图、ER图、数据库关系图、接口流程图、安全设计图 | 8-10 |
| 第5章 | 页面实现图、关键模块流程图、交互流程图、接口调用图 | 10-14 |
| 第6章 | 测试流程图、混淆矩阵、ROC曲线、指标对比图、性能图、测试截图 | 6-8 |

每张图必须满足：

1. 正文先引出或紧邻解释，不能孤立出现。
2. 图号按章连续，例如“图4.3”。
3. 图题置于图下，居中。
4. 图中文字清晰，中文不乱码。
5. 原理图、流程图、ER图默认使用黑白线条和白色或透明填充。
6. 系统截图必须是用户系统真实截图，不得用虚构界面冒充。
7. 同类图件遵循 `diagram-norm-cn`，画图前仍要联网查该图类标准。

## 表格与三线表

论文表格不少于6个时，优先覆盖以下内容：

1. 文献或技术方案对比表。
2. 功能需求表。
3. 非功能需求表。
4. 数据库表结构表。
5. 测试用例表。
6. 实验指标结果表。
7. 经济成本或进度计划表。

表题置于表上，居中；表号按章连续，例如“表5.2”。学术表格默认使用三线表：顶线、表头下横线、底线；通常不使用竖线。若学校模板要求普通网格表，按模板执行。

## 常见版式基线

没有学校模板时，采用以下基线并在交付说明中标明“通用排版”：

1. 页面：A4；页边距上2.5厘米、下2.5厘米、左2.8厘米、右2.5厘米。
2. 正文：宋体，小四，首行缩进2字符，1.5倍行距，两端对齐。
3. 一级标题：黑体，小三或三号，居中或左对齐按模板；章标题另起页。
4. 二级标题：黑体，四号或小四，左对齐。
5. 三级标题：黑体或宋体加粗，小四，左对齐。
6. 摘要标题、目录标题、参考文献标题按一级标题处理。
7. 图题、表题：宋体五号或小五，居中；图题在下，表题在上。
8. 页眉页脚、页码、封面、原创性声明以模板为准；没有模板时只做基础页码。
9. 段前段后不随意空行；标题与正文的间距用段前段后控制，不用连续空段。

## 交叉引用与Word域

论文中“如图4.3所示”“见表5.2”应优先使用 Word 题注和交叉引用域。自动交叉引用通常依赖 `SEQ`、`REF`、`PAGEREF` 等域代码。

处理要求：

1. 新建 docx 时，尽量为图题、表题和公式编号建立稳定题注。
2. 若使用 `python-docx` 生成静态文本编号，必须运行脚本检查编号连续性，并在交付中说明未使用 Word 动态域。
3. 若模板已有题注样式和交叉引用域，应保留并复制其模式，不要改成普通文本。
4. 修改图片、表格或章节顺序后，必须重新检查所有“图x.x”“表x.x”“式x.x”的引用。
5. 交付前设置文档打开时更新域；若本机没有 Word 或 LibreOffice，需提示用户打开 Word 后全选并更新域。

## 参考文献与正文引用

默认采用 GB/T 7714 顺序编码制：

1. 正文首次引用顺序必须从 `[1]` 递增。
2. 文末参考文献顺序必须与正文首次引用顺序一致。
3. 期刊文献尽量补全作者、题名、刊名、年、卷、期、页码、DOI。
4. 近三年、英文、期刊数量等指标按用户或学校要求检查。
5. 所有新增文献必须联网核验，不能只凭模型记忆写条目。

## 交付前硬性检查

对于用户要求的完整项目论文，默认至少检查：

1. 正文净字数是否不少于用户要求，常见为20000字。
2. 图片题注数量和嵌入图片数量是否不少于用户要求，常见为40张。
3. 表格数量和表题数量是否不少于用户要求，常见为6个。
4. 参考文献是否真实可查，正文是否全部引用。
5. 正文引用首次出现顺序是否递增。
6. 图号、表号是否按章连续，有无跳号、重号。
7. 图表是否在正文中被提及。
8. 正文样式是否满足字体、字号、缩进、行距、页边距要求。
9. 是否存在空泛表述、虚假实验结果、虚假截图或虚构数据。
