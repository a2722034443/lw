本地 DOCX 论文规范化助手使用说明

这个页面用来回答一个最实际的问题：你拿到一篇 WPS/Word 的论文 DOCX 后，应该怎样用当前项目完成读取、检查、保守修正、交叉引用链接和验证。

一句话理解

当前项目是一个本地运行的 DOCX 论文规范化工具。它不是在线写作机器人，而是帮你把论文文件拆开检查，再按模板规则做可回滚的格式修正。

标准流程是：

1

模板画像

读取学校模板或通用规则，生成格式 profile。

2

全量审查

检查结构、段落、图表、公式、引用、域代码。

3

保守修正

只改格式和可逆结构，不乱改论文事实内容。

4

引用链接

把正文 [1] 做成可跳到文末参考文献的链接。

5

二次验证

重新审查修正后的 DOCX，输出报告。

现在能做什么

读取与诊断

读取 DOCX 段落、表格、图片、节、样式、域代码和参考文献编号。

inspectdiagtemplate-inspect

格式审查

检查页边距、字体、字号、标题、正文、题注、三线表、段落属性。

auditcheck

保守修正

修正页边距、正文格式、标题格式、题注格式、三线表边框、段落设置。

fixformat

WPS 段落属性

支持对齐、大纲级别、左右缩进、首行缩进、段前段后、行距、孤行控制、中文换行习惯、标点溢出、中西文间距。

paragraph OOXML

参考文献跳转

给文末参考文献插入书签，把正文 [1] 包装成内部超链接，WPS/Word 可点击跳转。

link-references

验证与报告

输出 JSON 和 Markdown 报告。视觉验证有安全开关，避免 Word COM 隐藏弹窗卡住。

verifypipeline

推荐使用流程

第一次：先生成模板画像

项目里已有学校模板文件，但它是旧版 .doc。当前工具会安全降级生成通用 profile；如果你能用 Word 手动另存为 .docx，再跑一次会更准确。

python -m local_thesis_assistant.thesis_assistant template-inspect `
  "local_thesis_assistant\大连民族大学本科毕业设计说明书（论文）格式要求-2024.doc.doc" `
  --out local_thesis_assistant\outputs\template_profile.json `
  --rules-out local_thesis_assistant\outputs\template_rules.json

审查一篇论文

python -m local_thesis_assistant.thesis_assistant audit `
  your_thesis.docx `
  --profile local_thesis_assistant\outputs\template_profile.json `
  --json local_thesis_assistant\outputs\your_audit.json `
  --md local_thesis_assistant\outputs\your_audit.md

保守修正文档

不会覆盖原文件，会生成一个新的 DOCX。

python -m local_thesis_assistant.thesis_assistant fix `
  your_thesis.docx `
  local_thesis_assistant\outputs\your_thesis.fixed.docx `
  --profile local_thesis_assistant\outputs\template_profile.json

给参考文献做可点击跳转

前提是正文里有 [1]、[2]，文末参考文献也有 [1] xxx、[2] xxx。

python -m local_thesis_assistant.thesis_assistant link-references `
  local_thesis_assistant\outputs\your_thesis.fixed.docx `
  local_thesis_assistant\outputs\your_thesis.linked.docx `
  --json local_thesis_assistant\outputs\reference_links.json

二次验证

python -m local_thesis_assistant.thesis_assistant verify `
  local_thesis_assistant\outputs\your_thesis.linked.docx `
  --profile local_thesis_assistant\outputs\template_profile.json `
  --out-dir local_thesis_assistant\outputs\verify_your_thesis
视觉验证需要启动 Word COM，并且最好安装 pypdfium2。为了避免 WPS/Word 隐藏弹窗卡死，真正启动 Word 前需要显式设置 THESIS_ASSISTANT_ENABLE_WORD_COM_VISUAL=1。

一条命令跑完整流程

如果你只是想快速跑完整流程，可以用 pipeline。它会生成模板画像、审查修正前、修正、再验证。

python -m local_thesis_assistant.thesis_assistant pipeline `
  your_thesis.docx `
  local_thesis_assistant\outputs\your_thesis.fixed.docx `
  --template "local_thesis_assistant\大连民族大学本科毕业设计说明书（论文）格式要求-2024.doc.doc" `
  --out-dir local_thesis_assistant\outputs\pipeline_your_thesis

注意：pipeline 目前不会自动执行 link-references。建议你在修正后单独跑一次引用链接命令，因为引用链接属于正文引用结构增强，最好明确执行。

命令速查

命令作用常见输出
diag检查本机是否有 Word COM、LibreOffice 等转换能力。终端诊断结果
inspect快速读取 DOCX 的段落、表格、图片和节数量。终端摘要
template-inspect从学校模板生成规则画像。template_profile.json、template_rules.json
audit全量审查 DOCX。audit.json、audit.md
fix保守修正 DOCX 格式。新的 .docx
link-references把正文 [n] 链接到文末参考文献。新的 .docx、链接报告
verify二次审查修正后的 DOCX。verify.json、审查报告
pipeline串起模板、审查、修正、验证。流程输出目录

现在不能乱说已经完成的部分

已经比较稳的

  • DOCX 读取和基础结构统计。
  • 页边距、字体字号、段落属性和题注格式修正。
  • 表格三线表边框修正。
  • 正文引用到参考文献的内部跳转链接。
  • JSON / Markdown 报告。

仍然要谨慎的

  • 旧版 .doc 模板转换可能被 Word 隐藏弹窗卡住。
  • 视觉验证需要 Word COM 和 PDF 渲染依赖。
  • 不会自动核验参考文献真假。
  • 不会自动重写论文事实、实验结果和结论。
  • 项目符号、编号、多级编号还没有完整实现。

推荐你下一步怎么用

如果你现在有一篇自己的论文，最稳的做法是：

  1. 把论文 DOCX 放到 local_thesis_assistant/outputs/ 或任意你记得住的位置。
  2. 先跑 audit，只看报告，不改文件。
  3. 确认报告没误判后跑 fix，生成新文件。
  4. 如果正文有 [1] 引用，再跑 link-references。
  5. 最后跑 verify，看剩下的问题是不是内容类问题。
